| Данные                                                                       | Обоснование еобходимости защиты                                                                           | Инструменты защиты                                                                                         |
|------------------------------------------------------------------------------|-----------------------------------------------------------------------------------------------------------|------------------------------------------------------------------------------------------------------------|
| **Персональные данные пациента (PII)**<br>ФИО, телефон, email, дата рождения | Требования законодательства РФ (152-ФЗ), репутационные риски, риск нарушить права пациента на приватность | TLS при передаче, шифрование дисков БД, маскирование в API и логах, Keycloak, аудит доступа                |
| **Медицинские данные (MEDICAL)**<br>Диагнозы, анализы, заключения            | Медицинская тайна, максимальный уровень риска, прямые регуляторные санкции                                | TLS, шифрование at rest, field-level encryption, строгий ABAC, изоляция БД, аудит, запрет прямого экспорта |
| **Финансовые данные (FINANCIAL)**<br>Платежи, чеки, связи с 1С               | Финансовые риски, требования буху чета и налоговых органов                                                | TLS, шифрование at rest, маскирование идентификаторов, контроль доступов, аудит                            |
| **Документы пациентов (PDF/JPG)**<br>Сканы анализов, договоров               | Утечка файлов — самый частый сценарий инцидентов                                                          | Шифрование хранилища, DLP, контроль доступа, аудит скачиваний                                              |
| **Учётные данные пользователей**<br>Логины, токены, роли                     | Компрометация приводит к несанкционированному доступу                                                     | Keycloak, MFA, JWT с ограниченным TTL                                                                      |
| **Интеграционные данные (API)**<br>Запросы/ответы лаборатории                | Риск утечки MEDICAL через API                                                                             | TLS, schema validation, минимизация данных, аудит                                                          |
| **Аналитические данные (BI/ML)**                                             | Риск повторной идентификации пациентов                                                                    | Обезличивание, псевдонимизация, доступ только к авторизованной зоне                                        |
| **Логи и аудит**                                                             | Логи могут содержать чувствительные поля                                                                  | Маскирование логов, ограничение доступа, централизованное хранение                                         |
